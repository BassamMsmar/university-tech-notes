فكرة المادة

تطوير التطبيقات 1 يركز على الواجهة الأمامية Front-End، أي الجزء الذي يراه المستخدم ويتفاعل معه داخل المتصفح. الهدف ليس حفظ أوامر متفرقة، بل فهم كيف تتعاون HTML وCSS وJavaScript لبناء تجربة قابلة للاستخدام.

HTML
تبني هيكل الصفحة ومعناها.
CSS
تحدد الشكل، الألوان، المسافات، والتخطيط.
JavaScript
تضيف السلوك والتفاعل مع المستخدم.
Responsive Design
يجعل الصفحة مناسبة للجوال والتابلت والكمبيوتر.

ما هو تطبيق الويب؟

تطبيق الويب هو برنامج يعمل من خلال المتصفح مثل نظام الجامعة، المتجر الإلكتروني، أو لوحة متابعة الدرجات. لا يحتاج المستخدم غالبا إلى تثبيت التطبيق، بل يفتح رابطا ويبدأ الاستخدام.

تعريف: تطبيق الويب نظام برمجي يتكون غالبا من واجهة أمامية تعرض البيانات للمستخدم، وواجهة خلفية تعالج البيانات وتحفظها.
تقريب المفهوم: الواجهة الأمامية مثل صالة المطعم والقائمة التي يراها العميل، أما الواجهة الخلفية فهي المطبخ ونظام الطلبات. في نظام الجامعة، صفحة تسجيل الدخول Front-End، والتحقق من الرقم الجامعي في قاعدة البيانات Back-End.

HTML: بناء هيكل الصفحة

HTML ليست لغة برمجة كاملة، بل لغة ترميز تصف مكونات الصفحة: عنوان، فقرة، صورة، زر، نموذج، جدول، وغير ذلك. جودة HTML تؤثر على قابلية الوصول، محركات البحث، وسهولة صيانة المشروع.

  • Element: عنصر في الصفحة مثل p أو button.
  • Tag: الوسم المستخدم لكتابة العنصر مثل <h1>.
  • Attribute: خاصية تضيف معلومات للعنصر مثل href أو class.
  • Semantic HTML: استخدام عناصر ذات معنى مثل header وmain وsection.
<!DOCTYPE html>
<html lang="ar" dir="rtl">
<head>
  <meta charset="UTF-8">
  <title>نظام متابعة الطلاب</title>
</head>
<body>
  <header>
    <h1>نظام متابعة الطلاب</h1>
  </header>

  <main>
    <section>
      <h2>قائمة الطلاب</h2>
      <p>يعرض هذا القسم أسماء الطلاب المسجلين في المقرر.</p>
      <button>إضافة طالب</button>
    </section>
  </main>
</body>
</html>

CSS: تنسيق الصفحة

CSS مسؤولة عن الشكل: الألوان، الخطوط، المسافات، أماكن العناصر، وحجمها على الشاشات المختلفة. تخيلها كقواعد الهوية البصرية للموقع.

Box Model: طريقة حساب مساحة العنصر: المحتوى، الحشو padding، الحدود border، والهوامش margin.
body {
  font-family: Arial, sans-serif;
  background: #f6f8fb;
  color: #172033;
}

.student-card {
  background: white;
  border: 1px solid #d9e2ec;
  border-radius: 8px;
  padding: 16px;
  margin-bottom: 12px;
}

button {
  background: #0f766e;
  color: white;
  border: 0;
  padding: 10px 16px;
  border-radius: 6px;
}

JavaScript: إضافة التفاعل

JavaScript تجعل الصفحة تتفاعل مع المستخدم: عند الضغط على زر، تعبئة نموذج، إظهار رسالة، إخفاء عنصر، أو جلب بيانات من الخادم.

  • DOM: تمثيل صفحة HTML كأنها شجرة يمكن للغة JavaScript التعامل معها.
  • Event: حدث يحصل في الصفحة مثل click أو input.
  • Function: مجموعة أوامر تنفذ عند الحاجة.
<button id="saveBtn">حفظ</button>
<p id="message"></p>

<script>
  const saveBtn = document.querySelector("#saveBtn");
  const message = document.querySelector("#message");

  saveBtn.addEventListener("click", function () {
    message.textContent = "تم حفظ بيانات الطالب بنجاح";
  });
</script>

التصميم المتجاوب

التصميم المتجاوب يعني أن الصفحة تعمل بشكل جيد على الجوال والتابلت والكمبيوتر، دون أن يضطر المستخدم إلى التكبير أو السحب أفقيا.

قاعدة عملية: ابدأ بتصميم بسيط يعمل على الجوال، ثم وسعه للشاشات الكبيرة. هذه الطريقة تسمى Mobile First.
.layout {
  display: grid;
  grid-template-columns: 2fr 1fr;
  gap: 20px;
}

@media (max-width: 700px) {
  .layout {
    grid-template-columns: 1fr;
  }
}

مثال تطبيقي وتدريب

مثال: بطاقة طالب تظهر الاسم والتخصص وزر حالة التسجيل. 
<article class="student-card">
  <h2>أحمد محمد</h2>
  <p>التخصص: علوم الحاسب</p>
  <button id="statusBtn">عرض الحالة</button>
  <p id="statusText"></p>
</article>

<script>
  document.querySelector("#statusBtn").addEventListener("click", () => {
    document.querySelector("#statusText").textContent = "الحالة: مسجل في المقرر";
  });
</script>
تدريب: صمم صفحة "لوحة مقرراتي" تحتوي على ثلاث بطاقات لمقررات دراسية، وزر داخل كل بطاقة يظهر رسالة باسم المقرر، واجعل البطاقات متجاوبة مع الجوال.
