# ملخص مادة تطوير التطبيقات 2

## فكرة المادة

تطوير التطبيقات 2 ينتقل من واجهة المستخدم إلى الواجهة الخلفية Back-End. في هذه المادة يتعلم الطالب كيف يستقبل الخادم الطلبات، يعالجها، يتعامل مع قاعدة البيانات، ثم يرجع النتيجة للمتصفح. سنستخدم Python وDjango كنموذج عملي.

## أهداف التعلم

- فهم دور الواجهة الخلفية في تطبيقات الويب.
- استخدام Git وGitHub لإدارة نسخ المشروع.
- إنشاء بيئة افتراضية Python Virtual Environment.
- فهم بنية Django: Project, App, URL, View, Model, Template.
- إنشاء نموذج Model وربطه بقاعدة بيانات.
- تنفيذ مثال CRUD مبسط.

## 1. ما هي الواجهة الخلفية؟

الواجهة الخلفية هي الجزء الذي لا يراه المستخدم، لكنه يقرر ماذا يحدث خلف الشاشة: التحقق من تسجيل الدخول، حفظ البيانات، قراءة السجلات، إرسال البريد، وتطبيق قواعد الصلاحيات.

### تقريب المفهوم بأكثر من طريقة

- **نظام الجامعة**: الطالب يرى صفحة إدخال الرقم الجامعي، لكن الخادم يتحقق من الرقم وكلمة المرور.
- **المتجر الإلكتروني**: المستخدم يضغط "شراء"، والخادم يتحقق من المخزون، يحسب السعر، ويحفظ الطلب.
- **المكتبة**: الواجهة الأمامية هي مكتب الاستقبال، والواجهة الخلفية هي الفهارس والمخازن والنظام الداخلي.

## 2. Git وGitHub

Git يحفظ تاريخ المشروع. GitHub يستضيف المشروع على الإنترنت لتسهيل المشاركة والعمل الجماعي.

### أوامر أساسية

```bash
git init
git add .
git commit -m "initial version"
git status
git log --oneline
git push
```

### ملاحظة للطالب

استخدم Git كدفتر يوميات للمشروع. كل Commit يجب أن يصف خطوة واضحة: إضافة نموذج، إصلاح خطأ، أو تحسين واجهة.

## 3. البيئة الافتراضية venv

البيئة الافتراضية تعزل مكتبات كل مشروع. إذا كان لديك مشروع يستخدم Django 4 وآخر يستخدم Django 6، فلا يحدث تعارض بينهما.

### أوامر Windows

```bash
python -m venv .venv
.venv\Scripts\activate
pip install django
deactivate
```

### تقريب المفهوم

البيئة الافتراضية مثل حقيبة مستقلة لكل مقرر: أدوات مقرر قواعد البيانات لا تختلط بأدوات مقرر الشبكات.

## 4. لماذا Django؟

Django إطار عمل Web Framework بلغة Python يساعدنا على بناء تطبيقات ويب بسرعة وبشكل منظم. يوفر نظام مسارات، تعامل مع قاعدة البيانات، لوحة إدارة، حماية أمنية، ونظام قوالب.

### تعريفات مهمة

- **Framework**: إطار جاهز يعطيك قواعد وأدوات لبناء التطبيق.
- **Project**: الحاوية الكبرى لإعدادات الموقع.
- **App**: جزء مستقل داخل المشروع مثل الطلاب، المقررات، الدرجات.
- **ORM**: طبقة تسمح بالتعامل مع قاعدة البيانات باستخدام كود Python بدلا من SQL المباشر في كثير من الحالات.

## 5. إنشاء مشروع Django

## 6. معمارية MVT في Django

Django يستخدم نمط Model-View-Template:

- **Model**: يمثل البيانات، مثل جدول الطلاب.
- **View**: يستقبل الطلب ويقرر ماذا يرجع.
- **Template**: صفحة HTML تعرض البيانات.
- **URL**: يربط الرابط بالدالة المناسبة.

### مثال ذهني

طالب يريد رؤية درجاته:

1. يفتح رابط `/grades/`.
2. URL يوجه الطلب إلى View.
3. View يقرأ البيانات من Model.
4. Template يعرض الدرجات في صفحة HTML.

## 7. مثال Model

```python
# students/models.py
from django.db import models


class Student(models.Model):
    name = models.CharField(max_length=120)
    university_id = models.CharField(max_length=20, unique=True)
    major = models.CharField(max_length=80)
    created_at = models.DateTimeField(auto_now_add=True)

    def __str__(self):
        return f"{self.name} - {self.university_id}"
```

بعد تعديل النموذج:

```bash
python manage.py makemigrations
python manage.py migrate
```

## 8. مثال View وURL

## 9. لوحة الإدارة Admin

Django يوفر لوحة إدارة جاهزة لإضافة وتعديل البيانات.

```python
# students/admin.py
from django.contrib import admin
from .models import Student

admin.site.register(Student)
```

إنشاء مدير للنظام:

```bash
python manage.py createsuperuser
```

## 10. قاعدة البيانات SQLite

SQLite قاعدة بيانات ملفية بسيطة ومناسبة للتعلم والتجارب. في المشاريع الكبيرة يمكن استخدام PostgreSQL أو MySQL، لكن SQLite ممتازة كبداية لأنها لا تحتاج إعداد خادم مستقل.

### تقريب المفهوم

SQLite تشبه ملف Excel ذكي: موجود داخل المشروع، يحفظ الجداول، ويمكن نقله مع ملفات التطبيق.
