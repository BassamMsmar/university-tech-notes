مقدمة في الواجهة الخلفية (Back-End)

كما تم دراسته في المقرر السابق، تتكون تطبيقات الويب من واجهة أمامية (Front-End) وواجهة خلفية (Back-End). الواجهة الخلفية هي الجزء الذي لا يراه المستخدم، ولكنه العصب الأساسي الذي يقوم بمعالجة البيانات وتخزينها وتأمينها.

مثال واقعي: عند تسجيل الدخول إلى موقع الجامعة، الواجهة الأمامية هي النموذج الذي تكتب فيه الرقم الجامعي وكلمة المرور، أما الواجهة الخلفية فهي النظام الذي يتحقق من صحة هذه البيانات في قاعدة البيانات ويمنحك حق الدخول.

التحكم بالإصدارات (Git و GitHub)

يعتبر Git نظاماً للتحكم بالإصدارات يُستخدم لحفظ تاريخ وتطور المشروع، بحيث يمكن للمطور الرجوع إلى أي نسخة سابقة. أما GitHub فهو المنصة السحابية التي يتم رفع المشروع عليها للتعاون أو حفظه على الإنترنت.

  • Git: يمثل زر الانقاذ (حفظ) المستمر لتغيرات الشفرة المصدرية.
  • GitHub: يمثل الخزنة السحابية التي تحمي المشروع وتشاركه.

الأوامر الأساسية:

git init              // لإنشاء مستودع جديد في المشروع
git add .             // لإضافة كافة الملفات والتعديلات
git commit -m "msg"   // لتوثيق وحفظ التغييرات
git push              // لرفع التغييرات إلى الخادم السحابي

البيئة الافتراضية (Virtual Environment)

البيئة الافتراضية (venv) هي مساحة عمل معزولة لكل مشروع برمجي، تضمن عدم تداخل المكتبات وإصداراتها بين المشاريع المختلفة على نفس الجهاز.

على سبيل المثال، إذا كان لديك مشروع يستخدم Django إصدار 3 ومشروع آخر يستخدم الإصدار 5، ستحدث تعارضات إن لم يتم عزل كل مشروع ببيئته الخاصة.

أوامر البيئة الافتراضية:

python -m venv venv  // إنشاء بيئة افتراضية جديدة باسم venv
venv\Scripts\activate // تفعيل البيئة (في نظام ويندوز)
deactivate           // إيقاف عمل البيئة الافتراضية

لماذا Django ومميزاته؟

إطار Django هو إطار عمل بلغة Python جاهز ومزود بأدوات قوية تساعد على بناء تطبيقات الويب بسرعة وموثوقية.

  • يقلل كتابة الكود: يوفر حلولاً جاهزة للمشاكل الشائعة.
  • عالي الأمان: يحمي التطبيق تلقائياً من العديد من الثغرات الأمنية مثل حقن قواعد البيانات.
  • لوحة تحكم إدارية: يقوم بتوليد لوحة تحكم كاملة وجاهزة فورياً لإدارة محتوى الموقع.

شركات عالمية تعتمد على إطار Django في بنيتها التحتية مثل Instagram و Pinterest.

آلية عمل Django وتنظيم الملفات

يتكون المشروع في Django من مشروع رئيسي يضم إعدادات النظام، ويحتوي على عدة تطبيقات فرعية صغيرة (Apps) كل منها ينجز مهمة محددة.

معمارية MVT (Model - View - Template)

يعتمد إطار Django على هيكلية تنظيمية تفصل مكونات التطبيق كالتالي:

  • Model (النموذج): المكون المسؤول عن تمثيل البيانات المهيكلة والتعامل مع قاعدة البيانات.
  • View (العرض): المكون الذي يحتوي على المنطق البرمجي، فهو يستقبل طلب المستخدم ويعالج البيانات عبر استدعاء النموذج المناسب.
  • Template (القالب): يمثل ملفات HTML المسؤولة عن عرض واجهة المستخدم النهائية بعد تمرير البيانات إليها.
  • URL (المسارات): نظام التوجيه الذي يربط الرابط المطلوب بدالة الـ View المناسبة.

قاعدة البيانات وإدارة النظام

قاعدة البيانات SQLite3

يأتي Django محملاً بقاعدة بيانات SQLite3 افتراضياً. تتميز بكونها ملفية ولا تحتاج إلى برنامج تثبيت خوادم معقدة، مما يجعلها مثالية لبيئة التطوير والتعلم.

حساب المدير (Superuser)

للدخول إلى النظام كمستخدم إداري بكامل الصلاحيات، يتم إنشاء هذا الحساب باستخدام أمر برمجي. هذا الحساب يستطيع التحكم الكامل بالبيانات والمستخدمين من خلال لوحة تحكم Django الجاهزة.

أوامر Django الهامة: 
django-admin startproject projectname // لإنشاء مشروع جديد
python manage.py startapp appname     // لإنشاء تطبيق فرعي
python manage.py makemigrations       // لتسجيل تغييرات قاعدة البيانات
python manage.py migrate              // لتطبيق التغييرات على قاعدة البيانات
python manage.py createsuperuser      // لإنشاء مدير نظام
python manage.py runserver            // لتشغيل السيرفر ورؤية النتيجة
