# ملخص مادة تطوير التطبيقات 1

## فكرة المادة

تطوير التطبيقات 1 يركز على الواجهة الأمامية Front-End، أي الجزء الذي يراه المستخدم ويتفاعل معه داخل المتصفح. في هذه المادة يتعلم الطالب كيف تتحول الفكرة إلى صفحة ويب واضحة، من خلال HTML لبناء الهيكل، وCSS لتنسيق الشكل، وJavaScript لإضافة التفاعل.

## أهداف التعلم

- التمييز بين الواجهة الأمامية Front-End والواجهة الخلفية Back-End.
- بناء صفحة HTML صحيحة دلاليا.
- استخدام CSS لتنسيق الألوان، المسافات، التخطيط، والاستجابة للشاشات.
- استخدام JavaScript للتحكم في عناصر الصفحة والتفاعل مع المستخدم.
- فهم أساسيات التصميم المتجاوب Responsive Design.

## 1. ما هو تطبيق الويب؟

تطبيق الويب هو برنامج يعمل من خلال المتصفح مثل نظام الجامعة، المتجر الإلكتروني، أو لوحة متابعة الدرجات. لا يحتاج المستخدم غالبا إلى تثبيت التطبيق، بل يفتح رابطا ويبدأ الاستخدام.

### تعريف توضيحي

**تطبيق الويب**: نظام برمجي يتكون غالبا من واجهة أمامية تعرض البيانات للمستخدم، وواجهة خلفية تعالج البيانات وتحفظها.

### تقريب المفهوم بأكثر من طريقة

- **تشبيه المطعم**: الواجهة الأمامية مثل صالة المطعم والقائمة التي يراها العميل، أما الواجهة الخلفية فهي المطبخ ونظام الطلبات.
- **تشبيه نظام الجامعة**: صفحة تسجيل الدخول هي Front-End، والتحقق من الرقم الجامعي وكلمة المرور داخل قاعدة البيانات هو Back-End.
- **تشبيه السيارة**: لوحة القيادة والأزرار تمثل Front-End، والمحرك والحساسات تمثل Back-End.

## 2. HTML: بناء هيكل الصفحة

HTML ليست لغة برمجة بالمعنى الكامل، بل لغة ترميز Markup تصف مكونات الصفحة: عنوان، فقرة، صورة، زر، نموذج، جدول، وغير ذلك.

### تعريفات مهمة

- **Element**: عنصر في الصفحة مثل `p` أو `button`.
- **Tag**: الوسم المستخدم لكتابة العنصر مثل `<h1>`.
- **Attribute**: خاصية تضيف معلومات للعنصر مثل `href` أو `class`.
- **Semantic HTML**: استخدام عناصر ذات معنى مثل `header`, `main`, `section`, `footer`.

### مثال HTML

```html
<!DOCTYPE html>
<html lang="ar" dir="rtl">
<head>
  <meta charset="UTF-8">
  <title>نظام متابعة الطلاب</title>
</head>
<body>
  <header>
    <h1>نظام متابعة الطلاب</h1>
  </header>

  <main>
    <section>
      <h2>قائمة الطلاب</h2>
      <p>يعرض هذا القسم أسماء الطلاب المسجلين في المقرر.</p>
      <button>إضافة طالب</button>
    </section>
  </main>
</body>
</html>
```

### ملاحظة للطالب

إذا كان HTML هو العظام، فإن CSS هو الشكل الخارجي، وJavaScript هي الحركة والتفاعل.

## 3. CSS: تنسيق الصفحة

CSS مسؤولة عن الشكل: الألوان، الخطوط، المسافات، أماكن العناصر، وحجمها على الشاشات المختلفة.

### تعريفات مهمة

- **Selector**: يحدد العنصر المطلوب تنسيقه مثل `button` أو `.card`.
- **Property**: الخاصية المراد تغييرها مثل `color`.
- **Value**: قيمة الخاصية مثل `blue`.
- **Box Model**: طريقة حساب مساحة العنصر: المحتوى، الحشو `padding`، الحدود `border`، والهوامش `margin`.

### مثال CSS

```css
body {
  font-family: Arial, sans-serif;
  background: #f6f8fb;
  color: #172033;
}

.student-card {
  background: white;
  border: 1px solid #d9e2ec;
  border-radius: 8px;
  padding: 16px;
  margin-bottom: 12px;
}

button {
  background: #0f766e;
  color: white;
  border: 0;
  padding: 10px 16px;
  border-radius: 6px;
}
```

### تقريب المفهوم

CSS مثل قواعد الزي الرسمي في الجامعة: تحدد الشكل العام، الألوان، المسافات، وطريقة ظهور العناصر، لكنها لا تغير المعنى الأساسي للصفحة.

## 4. JavaScript: إضافة التفاعل

JavaScript تجعل الصفحة تتفاعل مع المستخدم: عند الضغط على زر، تعبئة نموذج، إظهار رسالة، إخفاء عنصر، أو جلب بيانات من الخادم.

### تعريفات مهمة

- **DOM**: تمثيل صفحة HTML كأنها شجرة يمكن للغة JavaScript التعامل معها.
- **Event**: حدث يحصل في الصفحة مثل click أو input.
- **Function**: مجموعة أوامر تنفذ عند الحاجة.

### مثال JavaScript

```html
<button id="saveBtn">حفظ</button>
<p id="message"></p>

<script>
  const saveBtn = document.querySelector("#saveBtn");
  const message = document.querySelector("#message");

  saveBtn.addEventListener("click", function () {
    message.textContent = "تم حفظ بيانات الطالب بنجاح";
  });
</script>
```

### تقريب المفهوم

JavaScript مثل موظف الاستقبال: ينتظر أن يقوم الطالب بإجراء معين، ثم يرد عليه برسالة أو ينقله للخطوة التالية.

## 5. التصميم المتجاوب Responsive Design

التصميم المتجاوب يعني أن الصفحة تعمل بشكل جيد على الجوال والتابلت والكمبيوتر، دون أن يضطر المستخدم إلى التكبير أو السحب أفقيا.

### مثال CSS للشاشات الصغيرة

```css
.layout {
  display: grid;
  grid-template-columns: 2fr 1fr;
  gap: 20px;
}

@media (max-width: 700px) {
  .layout {
    grid-template-columns: 1fr;
  }
}
```

### قاعدة عملية

ابدأ بتصميم بسيط يعمل على الجوال، ثم وسعه للشاشات الكبيرة. هذه الطريقة تسمى Mobile First.

## مثال تطبيقي صغير

المطلوب: بناء بطاقة طالب تظهر الاسم والتخصص وزر حالة التسجيل.

```html
<article class="student-card">
  <h2>أحمد محمد</h2>
  <p>التخصص: علوم الحاسب</p>
  <button id="statusBtn">عرض الحالة</button>
  <p id="statusText"></p>
</article>

<script>
  document.querySelector("#statusBtn").addEventListener("click", () => {
    document.querySelector("#statusText").textContent = "الحالة: مسجل في المقرر";
  });
</script>
```

## تدريب للطلاب

صمم صفحة بسيطة بعنوان "لوحة مقرراتي" تحتوي على:

- عنوان رئيسي.
- ثلاث بطاقات لمقررات دراسية.
- زر داخل كل بطاقة.
- عند الضغط على الزر تظهر رسالة باسم المقرر.
- اجعل البطاقات في عمود واحد على الجوال وثلاثة أعمدة على الشاشة الكبيرة.
