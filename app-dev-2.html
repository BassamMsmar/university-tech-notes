فكرة المادة

تطوير التطبيقات 2 ينتقل من واجهة المستخدم إلى الواجهة الخلفية Back-End. هنا يتعلم الطالب كيف يستقبل الخادم الطلبات، يعالجها، يتعامل مع قاعدة البيانات، ثم يرجع النتيجة للمتصفح.

ما هي الواجهة الخلفية؟

الواجهة الخلفية هي الجزء الذي لا يراه المستخدم، لكنه يقرر ماذا يحدث خلف الشاشة: التحقق من تسجيل الدخول، حفظ البيانات، قراءة السجلات، إرسال البريد، وتطبيق قواعد الصلاحيات.

تقريب المفهوم: في المتجر الإلكتروني يضغط المستخدم "شراء" في الواجهة الأمامية، لكن الواجهة الخلفية تتحقق من المخزون، تحسب السعر، تحفظ الطلب، ثم ترسل النتيجة.
API: واجهة تواصل بين الأنظمة. الواجهة الأمامية تطلب بيانات من API، والخادم يرجع استجابة مثل JSON أو صفحة HTML.

Git وGitHub

Git يحفظ تاريخ المشروع. GitHub يستضيف المشروع على الإنترنت لتسهيل المشاركة والعمل الجماعي. استخدم Commit لوصف خطوة واضحة لا لحفظ عشوائي.

git init
git add .
git commit -m "initial version"
git status
git log --oneline
git push

البيئة الافتراضية venv

البيئة الافتراضية تعزل مكتبات كل مشروع. إذا كان لديك مشروع يستخدم Django 4 وآخر يستخدم Django 6، فلا يحدث تعارض بينهما.

python -m venv .venv
.venv\Scripts\activate
pip install django
deactivate
تشبيه: البيئة الافتراضية مثل حقيبة مستقلة لكل مقرر؛ أدوات مقرر قواعد البيانات لا تختلط بأدوات مقرر الشبكات.

معمارية MVT في Django

Django يستخدم نمط Model-View-Template. الفكرة الأساسية هي فصل البيانات عن منطق المعالجة عن طريقة العرض.

المكوندورهمثال في نظام جامعة
Modelتمثيل البياناتجدول الطلاب
Viewاستقبال الطلب واتخاذ القرارعرض بيانات طالب
Templateإظهار HTML للمستخدمصفحة ملف الطالب
URLربط الرابط بالدالة/students/

أمثلة كود

Model لطالب

from django.db import models


class Student(models.Model):
    name = models.CharField(max_length=120)
    university_id = models.CharField(max_length=20, unique=True)
    major = models.CharField(max_length=80)
    created_at = models.DateTimeField(auto_now_add=True)

    def __str__(self):
        return f"{self.name} - {self.university_id}"
python manage.py makemigrations
python manage.py migrate
